- [GH-0134](GH-0134-schedule-d-datecomma-0001.md) — **Schedule D `Month DD, YYYY` Date incurred consumed whole**: a single-line Date incurred written `Month DD, YYYY` (e.g. `April 15, 2019`) used to fail `_FD_DATE_RE`'s `Month\s+\d{4}` month branch and anchor only on the bare-year fallback, leaking the `Month DD,` fragment into `creditor` and leaving just the year in `date_incurred` (silently-plausible corruption, distinct from #102's amount loss). Fixed by extending the month branch with an optional day-comma group (`(?:\d{1,2},\s+)?\d{4}`) so the full date matches and is consumed whole; the comma-less `Month YYYY` form and #102's wrapped-amount handling are preserved, and `_FD_DATE_RE` is used only in `_parse_schedule_d` so the blast radius is confined to Schedule D. No `SCHEMA_VERSION` bump (stays 8). (#134, omnibus #136)
