- [GH-0056](GH-0056-inspect-0001.md) — `openhouse inspect`: human accuracy review of parsed filings. A fourth verb samples reviewable (`parse_status: ok`) filings and shows each beside its source PDF in a small local web app to record a **precision/recall verdict** — surfacing silent recall failures (scanned PTRs that extract zero trades) the manifest can't. Verdict is a 2×2 over the sound/complete duality (entries *and* metadata) + optional entry magnitudes + a note, **snapshot-pinned** (a re-parse that changes a filing flags its label stale; `LABELS_SCHEMA_VERSION` is independent of the parsed-data `SCHEMA_VERSION`). Sampling is seeded/monotonic/stratified (`pdf_class` × is-PTR, `ceil(sample·n)` per cell). Stdlib `http.server` + `pdfplumber` (no new deps); **Svelte→static SPA via Vite, not SvelteKit**, bundle committed under `openhouse/inspect/static/` for zero-Node runtime. Scorecard JSON to stdout, residual to stderr. Filing mode only — trade mode + gold corrections, CI scoring, and metadata counts deferred.
